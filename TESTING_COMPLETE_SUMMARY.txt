═══════════════════════════════════════════════════════════════════════
  🔍 CACHING EXPLAINED
═══════════════════════════════════════════════════════════════════════

HOW IT WORKS:
1. First call → Cache MISS → API call → Store in cache
2. Subsequent calls → Cache HIT → Return instantly (<1ms)
3. Write operations → Auto-invalidate related cache entries
4. TTL expiry → Full refresh from API (not differential)

AUTOMATIC BEHAVIOR:
✅ Caching integrated into MCP server code
✅ No manual cache calls needed in tools
✅ GET requests: Check cache first, then API
✅ POST/PUT/DELETE: Invalidate related cache entries
✅ Different TTLs for different resource types:
   - Static data (plugins): 24 hours
   - Semi-static (repos): 30 minutes  
   - Dynamic (assets): 5 minutes
   - Real-time (scans): 1 minute

See CACHING_DEEP_DIVE.md for complete explanation!
