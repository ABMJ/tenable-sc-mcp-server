## Root Cause Analysis

### Investigation Process

1. **Read Implementation** (line 1009-1228 in server.py)
2. **Compared with Working Summary Tool** (line 877-1006)
3. **Identified Bug Location**: Lines 1201-1205

### The Bug

**Location**: `src/tenable_sc_mcp/server.py:1201-1205`

**Problem Code:**
```python
"summary": {
    "total_records": response.get("totalRecords"),      # ❌ response undefined
    "returned_records": response.get("returnedRecords"), # ❌ response undefined
    "start_offset": start_offset,
    "end_offset": end_offset,
    "has_more": int(response.get("totalRecords", 0)) > end_offset,  # ❌ response undefined
},
```

**Issue**: Variable `response` was never defined in the function scope. The code correctly extracted API response data into `api_response` and `inner_response` variables (lines 1167-1172), but then referenced non-existent `response` variable when building the summary.

### The Fix

**Added**: `response_data` variable to track the correct data source

**Fixed Code:**
```python
# Extract vulnerability data - handle nested response
api_response = result.get("response", {})
if isinstance(api_response, dict) and "response" in api_response:
    inner_response = api_response.get("response", {})
    vuln_data = inner_response.get("results", [])
    response_data = inner_response  # ✅ Store correct reference
else:
    vuln_data = api_response.get("results", [])
    response_data = api_response  # ✅ Store correct reference

# ... format vulnerabilities ...

"summary": {
    "total_records": response_data.get("totalRecords"),      # ✅ Fixed
    "returned_records": response_data.get("returnedRecords"), # ✅ Fixed
    "start_offset": start_offset,
    "end_offset": end_offset,
    "has_more": int(response_data.get("totalRecords", 0)) > end_offset,  # ✅ Fixed
},
```

**Change Summary:**
- Added `response_data` variable assignment in both conditional branches
- Changed all `response.get()` calls to `response_data.get()`
- Correctly handles both nested and flat API response structures
