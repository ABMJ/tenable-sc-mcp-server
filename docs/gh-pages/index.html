Overview

The Tenable.sc MCP Server is a production-ready Model Context Protocol (MCP) server that provides a thin, efficient adapter layer for the Tenable Security Center Plus REST API. Built with enterprise needs in mind, it offers advanced caching, multi-tenancy support, and comprehensive observability.

Key Features

⚡ High Performance

Multi-tier caching with Redis support, async operations, and connection pooling for maximum throughput.

🔒 Enterprise Security

API key authentication, OAuth 2.0, RBAC, audit logging, and comprehensive compliance features.

🏢 Multi-Tenancy

Connect to multiple Tenable.sc instances with isolated caching, metrics, and configuration.

📊 Observability

Structured logging, OpenTelemetry integration, real-time metrics, and performance monitoring.

🛡️ Reliability

Circuit breakers, rate limiting, retry logic, and graceful degradation for production resilience.

🐳 Container-Ready

Docker, Docker Compose, and Kubernetes support with Helm charts for easy deployment.

Quick Start

Docker Deployment

# Build the image
docker build -t tenable-sc-mcp:latest .

# Create config file
cat > ~/.tenable-sc-mcp.env <<'EOF'
TSC_URL=https://sc.example.com
TSC_ACCESS_KEY=your-access-key
TSC_SECRET_KEY=your-secret-key
TSC_VERIFY_SSL=true
EOF

# Run the server
docker run -d \
  --name tenable-sc-mcp \
  --restart unless-stopped \
  -p 0.0.0.0:8000:8000 \
  -v ~/.tenable-sc-mcp.env:/config/tsc.env:ro \
  tenable-sc-mcp:latest \
  --transport streamable-http \
  --host 0.0.0.0 \
  --port 8000 \
  --env-file /config/tsc.env \
  --allow-remote-hosts

Docker Compose

# Set user permissions
export LOCAL_UID=$(id -u)
export LOCAL_GID=$(id -g)

# Start services
docker compose up -d

# View logs
docker compose logs -f
