# Security Policy

This repository is community-maintained and is not an officially supported Tenable project.

## Supported Versions

Security fixes are prioritized for the latest released version.

## Reporting a Vulnerability

Please do not open public issues for vulnerabilities that could expose users or infrastructure.

To report a vulnerability:

1. Open a GitHub issue with minimal details and the title prefix `security:`.
2. Request a private contact channel in that issue.
3. Share full reproduction details only through the private channel.

Include:

- affected version/tag
- deployment mode (`stdio` or `streamable-http`)
- impact and exploitation conditions
- reproduction steps and any logs (redacted)

I will acknowledge the report and work on a fix/release as quickly as possible.
