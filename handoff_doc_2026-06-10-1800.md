## 🏗️ MAJOR ARCHITECTURE CHANGE: Unified Filters Dict Pattern

### Problem Statement

**Before v1.2.0**, tools used explicit filter parameters:

```python
# ❌ OLD PATTERN (DEPRECATED)
@mcp.tool()
def tsc_list_ips(
    repository: str | None = None,
    asset_group: str | None = None,
    # 55+ explicit filter parameters
    asset_criticality: str | None = None,
    vpr_score: str | None = None,
    aes_score: str | None = None,
    severity: str | None = None,
    exploit_available: str | None = None,
    # ... 50+ more parameters
):
    # Manual mapping - 100+ lines of boilerplate
    filters = build_filters(
        asset_criticality=asset_criticality,
        vpr_score=vpr_score,
        # ... repeat all 55 parameters
    )
```

**Maintenance Burden:**
- 100+ lines of parameter declarations per tool
- Adding a new filter = updating 4 tools × 3 locations = 12+ edits
- High duplication, prone to inconsistencies
- Does not scale to 25 planned tools

### Solution: Unified Filters Dict

**After v1.2.0**, all tools use a single `filters` dict:

```python
# ✅ NEW PATTERN (v1.2.0+)
@mcp.tool()
def tsc_list_ips(
    repository: str | None = None,
    asset_group: str | None = None,
    ip: str | None = None,
    include_details: bool = False,
    # Unified filters dict - provides access to ALL 55+ filters
    filters: dict[str, Any] | None = None,
) -> dict[str, Any]:
    """
    [Tool description]
    
    Args:
        filters: Optional dict of filter parameters.
            For complete filter reference: tenable-sc://filters/reference
            
            Common filters:
                asset_criticality: ACR range (e.g., "7-10")
                vpr_score: VPR range (e.g., "7-10")
                severity: critical/high/medium/low/info
                exploit_available: Yes/No
                # ... 8-10 most common filters
    """
    # Extract filters dict (default to empty)
    filter_dict = filters or {}
    
    # Build filters - NO MANUAL MAPPING
    filter_list = build_filters(**filter_dict)
```

### Benefits

**For Maintainers:**
- ✅ 5 lines instead of 100+ per tool
- ✅ Add new filter = 1 edit to `COMMON_FILTERS` dict (zero tool edits)
- ✅ Zero duplication across tools
- ✅ Scales to 25+ tools effortlessly

**For Users:**
- ✅ Consistent interface across all tools
- ✅ All 55+ filters available everywhere
- ✅ One learning curve for all tools
- ✅ Better error messages with validation

**For LLMs:**
- ✅ Simpler function signatures
- ✅ Fewer parameters to understand
- ✅ Clear filter documentation via MCP resource

### Design Decision: Why NOT **kwargs?

**MCP Protocol Limitation**: MCP requires explicit JSON Schema for all parameters. `**kwargs` is treated as a literal parameter name `"kwargs"` by MCP, not as variable expansion.

**What Happened with Tool 5:**
```python
# BEFORE (BROKEN with MCP)
def tsc_list_vulns_by_cve(cve_id: str, **kwargs: Any):
    filters = build_filters(**kwargs)

# User calls:
tsc_list_vulns_by_cve("CVE-2021-44228", asset_criticality="8-10")

# MCP wraps as:
{"cve_id": "CVE-2021-44228", "kwargs": {"asset_criticality": "8-10"}}

# build_filters() receives:
kwargs = {"kwargs": {"asset_criticality": "8-10"}}

# Validation fails:
WARNING: Unknown filter parameters will be ignored: kwargs

# Result: All filters silently ignored!
```

**Solution**: Use explicit `filters: dict` parameter that MCP can properly serialize.

## 🐛 CRITICAL BUG FIX: Tool 5 MCP **kwargs Limitation

### Root Cause Analysis

**MCP Protocol Limitation**: MCP requires explicit JSON Schema for all parameters. Python's `**kwargs` dynamic parameter expansion is NOT supported by JSON Schema.

**What MCP Does:**
1. Reads function signature via introspection
2. Generates JSON Schema for parameters
3. Treats `**kwargs` as literal parameter name `"kwargs"` (not variable expansion)
4. Client sends: `{"cve_id": "...", "kwargs": {"asset_criticality": "8-10"}}`
5. Python function receives: `kwargs = {"kwargs": {"asset_criticality": "8-10"}}`

### Symptom

**User reported**: Tool 5 ACR filters don't work
```python
# User tries:
tsc_list_vulns_by_cve("CVE-2021-44228", asset_criticality="8-10")

# Expected: Returns only IPs with ACR 8-10
# Actual: Returns ALL 19 IPs (filter ignored)
```

**Logs showed:**
```
WARNING: Unknown filter parameters will be ignored: kwargs
```

### Fix

**Changed Tool 5 signature:**
```python
# BEFORE (BROKEN)
def tsc_list_vulns_by_cve(cve_id: str, **kwargs: Any):
    filters = build_filters(**kwargs)  # kwargs = {"kwargs": {...}}

# AFTER (FIXED)
def tsc_list_vulns_by_cve(
    cve_id: str,
    filters: dict[str, Any] | None = None,
):
    filter_dict = filters or {}
    filter_list = build_filters(**filter_dict)  # Correct unpacking
```

**Result**: ACR filters now work correctly in Tool 5 (verified by user testing)

### Lesson Learned

**FOR ALL MCP TOOLS:**
- ❌ **NEVER** use `**kwargs` in MCP tool signatures
- ✅ **ALWAYS** use explicit `dict` type for dynamic parameters
- ✅ Document this limitation in DESIGN_PRINCIPLES.md

**Impact**: This pattern is now MANDATORY for all 20+ future tools
